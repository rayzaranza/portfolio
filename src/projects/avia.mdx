avia é um aplicativo web para organizar tarefas por projeto. Criei o produto do design ao deploy para praticar front-end em um fluxo completo, passando por interface, autenticação, dados, testes e publicação.

Usei o projeto para estudar React 19, Supabase Auth, Tailwind com tokens próprios, testes unitários e deploy contínuo. Todo o código foi escrito por mim, com pesquisas e revisões pontuais usando Perplexity e Claude. Todo o código fonte está disponível no repositório <Link isExternal href="https://github.com/rayzaranza/avia">github.com/rayzaranza/avia</Link> e o app está em produção em <Link isExternal href="https://avia.rayzaranza.com.br">avia.rayzaranza.com.br</Link>.

<Video src="https://res.cloudinary.com/dulmlipof/video/upload/v1776884591/avia-demo_l4w3d7.mp4" />

## Funcionalidades

- Autenticação com email, Google e GitHub
- Criação de projetos
- Criação e conclusão de tarefas por projeto
- Exclusão de projetos e tarefas com opção de desfazer

## Processo

### Design

No Figma, explorei alguns estilos possíveis para definir a linguagem visual do app. Chegando em algo "blocado" e "tátil", com sombras duras e elementos em camadas. A ideia era que os elementos interativos seguissem o estilo `blocky`, com algumas variações, como `blocky-inset` para inputs e estado ativo de botões, simulando uma "indentação" na tela. A fonte é a <Link isExternal href="https://github.com/scfried/soft-type-jersey">Jersey 20</Link> e os ícones são da <Link isExternal href="https://github.com/hackernoon/pixel-icon-library">Pixel Icon Library</Link>.

Com essa linguagem definida já era possível criar a base do app. Criei os design tokens como variáveis no Figma e criei o kit de componentes para começar a desenhar as principais telas. Como o foco do projeto é desenvolver o front-end, decidi pular a etapa de desenhar os fluxos e protótipos interativos no Figma.

### Front-end

Com os tokens, componentes e layouts fechados já era possível começar o front-end. Iniciei um projeto em React e Typescript com Vite, usando Tailwind para o CSS. Levei os tokens do Figma como um tema no Tailwind para manter cores, espaçamentos e estados visuais consistentes, removendo os tokens predefinidos que não seriam usados. Aproveitei também para usar cores em OKLCH no CSS tornando mais intuitivo manipular as cores e expandir a paleta.

Também criei classes utilitárias para o estilo `blocky` pra usar em elementos interativos, já incluindo as cores, sombras, interações, etc.

### Autenticação

Usei Supabase Auth para cadastro por email e login com Google e GitHub. Para manter o foco no front-end, não criei um backend próprio: consumi a REST API do Supabase direto pela aplicação.

<ProjectImage
  src={authCodeImage}
  alt="Funções para autenticação com Supabase"
  caption="Funções para autenticação com Supabase"
/>

### Rotas

Para criar a navegação interna do app, usei o TanStack Router para criar rotas usando os arquivos criados na pasta `src/routes/`, com isso também fui capaz de criar rotas protegidas apenas para usuários logados e usar os loaders para carregar os dados com a API do Supabase.

<ProjectImage
  src={routesCodeImage}
  alt="Funções para autenticação com Supabase"
  caption="Arquivos de rota com TanStack Router"
/>

### Interações

Para as interações, principalmente a de marcar uma tarefa como concluída usei os hooks `useOptimistic` e `useTransition` do React 19 para evitar o delay de clicar e esperar a requisição terminar pra mostrar o estado atualizado.

<ProjectImage
  src={optimistToggleImage}
  alt="Uso dos hooks para mutação otimista da interface"
  caption="Uso dos hooks para mutação otimista da interface"
/>

Também implementei soft delete com opção de desfazer, reduzindo o impacto de exclusões acidentais. Usei o Sonner para gerenciar a lógica dos toasts.

<ProjectImage
  src={softDeleteImage}

### Testes

Usei Testing Library, Vitest e JSDOM para escrever testes unitários básicos. O objetivo não foi cobrir o app inteiro, mas entender melhor como testar componentes e fluxos de interface.

<ProjectImage
  src={testsImage}

Pra hospedar o app em produção criei um Worker no Cloudflare, apontando para um domínio próprio, e integrei com GitHub para fazer deploy contínuo.

## Aprendizados

- Conduzir um produto simples do design ao deploy
- Criar e aplicar tokens com Tailwind
- Integrar autenticação com Supabase Auth
- Consumir a REST API do Supabase no front-end
- Aplicar mutação otimista com React 19
- Criar soft delete com opção de desfazer
- Escrever testes unitários básicos
- Configurar deploy contínuo com Cloudflare e Wrangler
